Спецификация API для проекта SFMShop

1. GET /products
   Описание: Получение списка всех товаров
   Метод: GET
   Параметры: нет
   Тело запроса: нет

   Статусы ответа:
   200 OK - список товаров получен
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   GET /products HTTP/1.1
   Host: sfmshop.com

   Пример ответа:
   HTTP/1.1 200 OK
   Content-Type: application/json
   [{"id": 1, "name": "Ноутбук", "price": 50000}]

2. GET /products/{id}
   Описание: Получение товара по ID
   Метод: GET
   Параметры: id - идентификатор товара
   Тело запроса: нет

   Статусы ответа:
   200 OK - товар найден
   404 Not Found - товар не найден
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   GET /products/1 HTTP/1.1
   Host: sfmshop.com

   Пример ответа (найден):
   HTTP/1.1 200 OK
   Content-Type: application/json
   {"id": 1, "name": "Ноутбук", "price": 50000}

   Пример ответа (не найден):
   HTTP/1.1 404 Not Found
   Content-Type: application/json
   {"error": "Товар с id=1 не найден"}

   3. POST /products
   Описание: Создание нового товара
   Метод: POST
   Параметры: нет
   Тело запроса:
   {
     "name": "Название товара",
     "price": 50000
   }

   Статусы ответа:
   201 Created - товар создан
   400 Bad Request - неправильные данные
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   POST /products HTTP/1.1
   Host: sfmshop.com
   Content-Type: application/json
   {"name": "Ноутбук", "price": 50000}

   Пример ответа:
   HTTP/1.1 201 Created
   Content-Type: application/json
   {"id": 1, "message": "Товар создан"}

   4. PUT /products/{id}
   Описание: Обновление товара по ID
   Метод: PUT
   Параметры: id - идентификатор товара
   Тело запроса:
   {
     "name": "Новое название",
     "price": 45000
   }

   Статусы ответа:
   200 OK - товар обновлён
   400 Bad Request - неправильные данные
   404 Not Found - товар не найден
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   PUT /products/1 HTTP/1.1
   Host: sfmshop.com
   Content-Type: application/json
   {"name": "Ноутбук обновлённый", "price": 45000}

   Пример ответа:
   HTTP/1.1 200 OK
   Content-Type: application/json
   {"id": 1, "name": "Ноутбук обновлённый", "price": 45000}

   5. DELETE /products/{id}
   Описание: Удаление товара по ID
   Метод: DELETE
   Параметры: id - идентификатор товара
   Тело запроса: нет

   Статусы ответа:
   200 OK - товар удалён
   404 Not Found - товар не найден
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   DELETE /products/1 HTTP/1.1
   Host: sfmshop.com

   Пример ответа:
   HTTP/1.1 200 OK
   Content-Type: application/json
   {"message": "Товар удалён"}

   6. GET /orders
   Описание: Получение списка всех заказов
   Метод: GET
   Параметры: нет
   Тело запроса: нет

   Статусы ответа:
   200 OK - список заказов получен
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   GET /orders HTTP/1.1
   Host: sfmshop.com

   Пример ответа:
   HTTP/1.1 200 OK
   Content-Type: application/json
   [{"id": 5, "user_id": 1, "product_id": 2, "quantity": 1}]

7. POST /orders
   Описание: Создание нового заказа
   Метод: POST
   Параметры: нет
   Тело запроса:
   {
     "user_id": 1,
     "product_id": 2,
     "quantity": 1
   }

   Статусы ответа:
   201 Created - заказ создан
   400 Bad Request - неправильные данные
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   POST /orders HTTP/1.1
   Host: sfmshop.com
   Content-Type: application/json
   {"user_id": 1, "product_id": 2, "quantity": 1}

   Пример ответа:
   HTTP/1.1 201 Created
   Content-Type: application/json
   {"id": 5, "message": "Заказ создан"}

8. GET /users/{id}/orders
   Описание: Получение заказов пользователя
   Метод: GET
   Параметры: id - идентификатор пользователя
   Тело запроса: нет

   Статусы ответа:
   200 OK - заказы получены
   404 Not Found - пользователь не найден
   500 Internal Server Error - ошибка сервера

   Пример запроса:
   GET /users/1/orders HTTP/1.1
   Host: sfmshop.com

   Пример ответа:
   HTTP/1.1 200 OK
   Content-Type: application/json
   [{"id": 5, "product_id": 2, "quantity": 1}]
